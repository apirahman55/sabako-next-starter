## About The Project

Our mission is to accelerate the development process of fullstack applications by providing a comprehensive and easy-to-use boilerplate. Developers no longer need to start projects from scratch, saving time and increasing efficiency.

Here's why:
* Creating a fullstack project from scratch takes a long time and is inefficient. A boilerplate provides a ready-made basic structure, so developers can focus directly on developing the features and logic of the application.
* A boilerplate helps developers maintain consistent code style and project structure. This can make the codebase easier to read and understand for other developers.
* A standardized boilerplate can help developers collaborate more easily, as they do not need to learn different project structures.

Of course, no one boilerplate will serve all projects since your needs may be different. So I'll be adding more in the near future. You may also suggest changes by forking this repo and creating a pull request or opening an issue. Thanks to all the people have contributed to expanding this boilerplate!

### Prerequisites

This is an example of how to list things you need to use the software and how to install them.

* Make sure your `npm` been latest version
  ```sh
  npm install npm@latest -g
  ```
* This project using `pnpm` you'll need install globally
  ```sh
  npm install -g pnpm
  ```
* Make sure your commit fulfill if you want to contribute [this requirement](https://medium.com/@naandalist/creating-a-git-commit-message-convention-for-your-team-acb4b3edfc44)

### Installation

_Below is an example of how you can instruct your audience on installing and setting up your app. This template doesn't rely on any external dependencies or services._

1. Clone the repo
   ```sh
   git clone https://github.com/apirahman55/sabako-next-starter.git
   ```
2. Install NPM packages
   ```sh
   pnpm install
   ```
3. Copy a `.env.example` file into `.env.local` in the root directory and include your database connection details and other environment variables.

4. Run development server
    ```sh
    pnpm dev
    ```
